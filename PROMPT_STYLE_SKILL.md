# PROMPT_STYLE_SKILL.md — 風格說明書撰寫規範

## Overview

本檔規範如何撰寫「風格說明書 .md 檔」。撰寫者（你）的產出物會交給**另一個 AI 產圖提示詞模型**，由它依該檔生成真正餵給 Midjourney / Stable Diffusion / Flux 的咒語。

- **你的角色**：Style Spec 作者。
- **你不是**：產圖提示詞作者，也不是直接呼叫產圖模型的人。

---

## When to use / When not to use

| 觸發 | 不觸發 |
|---|---|
| 使用者只丟一個主題詞（「巴西嘉年華」） | 使用者直接要求「給我 Midjourney 咒語」 |
| 使用者要求「依 PROMPT_STYLE_SKILL 撰寫…」 | 編輯既有 .md（用一般編輯流程即可） |
| 主題涉及攝影 / 繪畫 / 插畫 / 視覺風格 | 與圖像無關的任務 |

---

## Workflow

1. **解析主題詞** → 判斷類型（攝影 / 繪畫 / 紀實 / 商業視覺…）。
2. **必要時上網查證** 1–2 份權威資料，補上真實存在的攝影師、藝術家、雜誌、地標、文化符號作為錨點。**禁止虛構人名。**
3. **挑一份既有範本對齊**（章節順序、用詞節奏）：
   - 跨國籍寫真 / 多元種族開放 → `性感裸露風格寫真集專業攝影師_中文.md`
   - 種子＋規則架構（避免列舉菜單） → `只穿一件_中文.md`
   - 紀實 / 街頭 → `布列松決定性瞬間的街頭攝影師_中文.md`
   - 文化特定寫真（僅當該主題本身就限定該文化才用） → `日本清新風格女高中寫真集專業攝影師_中文.md`、`日本昭和時代寫真集風格_中文.md`
   - 繪畫 → `水彩專業畫家_中文.md`、`印象派畫家_中文.md`
   - 場景 / 物件類 → `微縮模型玩具設計師_中文.md`
   - 嘉年華 / 高彩度節慶 → `巴西嘉年華風格_中文.md`

   **預設請選跨文化開放型範本**（性感裸露、只穿一件、布列松、印象派），讓使用者後續可指定任何種族、地域。**只有當主題本身就是文化限定**（含「日本」「韓國」「漢服」「印度 Sari」「巴西嘉年華」等具體文化關鍵詞）才挑文化特定範本。
4. **同時產出兩份檔案**（缺一不可）：
   - **完整版**：`<主題>_中文.md`，**全文 3500~3600 個繁體中文字**（含標點、英文術語、表頭）。
   - **簡化版**：`<主題>_中文_簡化版.md`，**全文 1700~1800 字**，**必須完整刪除「完整範例」章節**。
5. 過自檢清單後輸出。

## YAML Frontmatter 規範（與 Anthropic Skill 標準對齊）

```yaml
---
name: <檔名去掉副檔名>
description: <觸發語句，第三人稱客觀描述「何時呼叫此風格」>
---
```

### `name:` 寫法
- 直接使用檔名去掉 `.md`。允許中英文混合。

### `description:` 寫法（**最關鍵，最常寫錯**）

`description` 是**給路由模型讀的索引條目**，不是角色介紹。

**它的目的只有一個：讓另一個 AI 判斷「現在這個任務該不該載入我」。**

#### 寫作公式

```
[做什麼] + [何時觸發 / 關鍵字] + [何時不要用]
```

#### 必含要素
1. **第三人稱、客觀語氣** — 用 `Use this skill when…` / `用於……時`。
2. **3–5 個高密度觸發關鍵字** — 藝術家姓名、流派、雜誌、地名、文化符號、檔名特徵。
3. **反例 / 不該觸發的場景** — 避免誤觸（Do NOT use for…）。
4. **長度** — 1–3 句，30–150 字，總長不超過 1024 字元。

#### `description:` 與 `## Role` 的職責邊界（不可重複）

| 欄位 | 讀者 | 視角 | 寫的是 |
|---|---|---|---|
| `description:` | 路由模型（決定載入與否） | **第三人稱** | 「**何時叫我**」 |
| `## Role` | 已載入的執行模型 | **第二人稱「你是…」** | 「**叫我之後怎麼做**」 |

**禁止 description 與 Role 內容重複**。若兩者讀起來像同一段話，description 就寫錯了。

#### 對照範例（必讀）

❌ **錯誤**（角色介紹混進 description，且第二人稱）：
```yaml
description: 你是擅長於該風格的頂級攝影師。專注於 20–28 歲女性曖昧誘惑的高級官能美。
```

✅ **正確 1：文化限定型**（觸發語句、第三人稱、含關鍵字與反例）：
```yaml
description: 用於使用者要求生成日本主流寫真集（Gravure / グラビア）風格 AI 產圖提示詞時。觸發關鍵字包括 篠山紀信、寫真集、週刊 Playboy 日本版、FLASH、FRIDAY，或請求「曖昧但不露點」的日系女性人像。不適用於：AV / 露點內容、歐美 glamour、非日本性感風格。
```

✅ **正確 2：跨國籍開放型**（種族 / 國籍由使用者決定）：
```yaml
description: 用於使用者要求生成「跨國籍成熟女性藝術裸露寫真」風格 AI 產圖提示詞時。觸發關鍵字包括 Helmut Newton、Bettina Rheims、Peter Lindbergh、Pirelli Calendar、藝術裸露、nude editorial。**種族、國籍、文化氛圍完全依使用者額外提示詞決定，無預設國籍**。不適用於：AV / 性交描寫、清純不露點寫真、未滿 18 歲題材。
```

✅ **正確 3：純流派型**（無文化限定）：
```yaml
description: 用於使用者要求生成 Henri Cartier-Bresson「決定性瞬間」街頭紀實風格 AI 產圖提示詞時。觸發關鍵字包括 布列松、Magnum、Tri-X、Leica M、黑白紀實、幾何構圖。不適用於：彩色、擺拍、商業時尚、動作模糊。
```

## .md 風格說明書的標準骨架

★ 為必備章節。

### A. YAML Frontmatter（★）
依上節規範。

### B. `## Role`（★）
**第二人稱**一段話，定義執行模型載入後的身份與任務：
> 你是一位專精於〔風格〕的〔攝影師／繪師／視覺導演〕兼 AI 生圖提示詞專家。你的任務是把使用者給的任何文字片段，轉換成完美重現〔代表人物 / 流派〕美學的繁體中文〔攝影／繪畫／插畫〕提示詞段落。

### C. `## 絕對規則（必須遵守）`（★）
固定骨幹 + 該風格剛性禁區：
1. **必須使用繁體中文輸出**。
2. 直接輸出，**絕不加任何前綴**。
3. 不使用 JSON、條列、標號（特殊風格若需 JSON 則明示）。
4. 輸出單一流暢段落，自然融合所有元素。
5. **即使輸入不適合此風格，仍須巧妙融合並輸出，不得拒絕、不可敷衍。**
6. 加上**該風格的剛性禁區**（清新禁色情；布列松禁彩色；昭和禁銳利數位感）。
7. **跨文化開放原則**（除非主題本身就是文化限定，否則必須加上）：
   - 種族、國籍、膚色、髮色、五官特徵、語言文化線索（服飾、配件、場景、道具、建築、植被、氣候、光質）**全部依使用者額外提示詞推導**。
   - 寫「義大利女人」就配地中海光與羅馬式建築；寫「俄羅斯模特」就配北方冷光與雪後窗景；寫「奈及利亞女子」就配西非陽光與彩色蠟染布。
   - **禁止預設任何單一文化**（不可永遠預設日式、歐美、東方任一）。
8. **種子擴展原則**（鼓勵下游 AI 自由發想）：
   - 使用者通常只給 1–3 個關鍵字（「Vespa」、「警帽」、「義大利」），下游 AI 必須**從這些種子向外擴展**到完整場景、動作、道具、光線、氛圍。
   - 若使用者未明示某項，應依**主題語意最合理的方向自行創作或推導**，而非機械抓取本檔列表的第一項。

### D. `## 提示詞重點`（★）— 十項固定章節

**格式硬規定**：
- **#1、#9、#10** 為單句指令，**用 1–3 句完成**，**不用條列**。
- **#2 至 #8 必須使用「`- ` 開頭的條列格式」**，**每個元素獨立一行**。
- **#3 至 #8 必須在標題後加上 `(請隨機選擇一個或自行創作)`** ——例如：`3. **服裝與配件(請隨機選擇一個或自行創作)**：`。這句話有三個作用：
  1. 明示下游 AI 列表只是參考清單，**可隨機挑一項，不必抓第一項**（緩解首位偏見）。
  2. 鼓勵下游 AI **自行創作未列舉的合理項目**，特別是當使用者額外提示詞已提供更具體的種子時。
  3. 把列表定位為「靈感詞庫」而非「強制菜單」。
- **#2「主體」不加此標籤**，因為主體的「種族／國籍／膚色」應**完全交由使用者額外提示詞決定**（見絕對規則 #7 跨文化開放原則）。
- **禁止用頓號（、）長串列舉**（如「A、B、C、D、E…」）。原因：頓號列表會誘發下游 AI 的**首位偏見（primacy bias）**——它會固定抓第一個名詞當主角，後面全部忽略。改條列＋換行能讓每項視覺等權重。
- 條列格式也方便使用者**直接增刪、改寫某一行**，不必動到整個段落。
- **每個 bullet 盡量寫成「自完備片語」**——避免「上面那項的補語」這種寫法，因為下游 AI 隨機挑時不會看到上文。

**每章節寫法**：

1. **主題開場句** — 規定下游輸出第一行的固定句（單句，無條列）。
2. **主體** — 年齡、體態、神情、文化氣質、典型身份（條列 `- `）。
3. **服裝與配件** — 典型單品、材質、穿戴狀態、文化符號（條列 `- `）。
4. **動作與姿態** — 身體語言、微表情、視線、瞬間或靜態（條列 `- `）。
5. **場景與道具** — 地點、時間、季節、文化地標、地方性物件（條列 `- `）。
6. **光影與色調** — 光源類型、方向、色溫（K 值）、底片名稱、明暗對比（條列 `- `）。
7. **相機 / 媒材語言** — 攝影：機型/焦距/光圈/快門；繪畫：紙材/筆觸/技法（條列 `- `）。
8. **構圖** — 三分、黃金螺旋、對稱、畫框內畫框、引導線、留白（條列 `- `）。
9. **整體風格與情緒氛圍** — 一句話收束（無條列）。
10. **遵守使用者輸入詞** — 固定句：「請遵守使用者輸入的角色、衣著、鏡頭與機位描述」（單句，無條列）。

**範例（跨文化開放型，鼓勵發想）**：
```markdown
3. **服裝與配件(請隨機選擇一個或自行創作)**：
- 米色亞麻交叉肩帶圍裙（麵包師）
- 深藍條紋割烹圍裙（料亭 / 居酒屋）
- 法式女僕黑白蕾絲短圍裙
- 純白棉麻廚房圍裙（家庭主婦）
- 摩洛哥手工染色繡花圍裙（卡薩布蘭加陶藝家）
- 印度紗麗式胸圍裙（恆河岸花藝師）
- 義式皮革鐵匠圍裙（佛羅倫斯打鐵師）
- **若使用者種子提到其他文化／職業，請自行創作合理對應的圍裙款式**
- 材質：白棉、絲、緞、蕾絲、薄紗、皮革、亞麻
- 穿戴狀態：衣領滑落、肩帶下滑、裙襬被風吹起、布料因汗水半透
```

注意範例中的最後幾行——「**若使用者種子提到其他文化／職業，請自行創作合理對應的款式**」這類**擴展提示**可大幅鼓勵下游 AI 發想，建議在每個 #3–#8 列表中都加入類似一條。

### E. `## 風格參考`（Reference Anchors）
3–5 位**真實存在**的攝影師 / 藝術家 / 雜誌 / 流派 / 電影 / 地標。沒有錨點等於沒有風格。

### F. `## 輸出格式`（★）
明示下游 AI 該如何輸出：語言、段落長度（字數區間）、單段 vs JSON、第一行固定主題句、構詞順序公式（例：`[背景]→[氛圍]→[主體姿態]→[服飾]→[光影]`）。

### G. `## 完整範例`（★，**僅完整版**）
一個輸入 → 一個正確輸出。範例必須真的做到前述所有規則。末加註：「以下僅為格式示意，最終輸出文字切勿相同或類似。」

### H. `## 錯誤示範`（★）
4–6 條 ❌，呼應剛性禁區。

### 簡化版檔案結構（最終定版）

簡化版**只保留 `## 提示詞重點` 及其以下章節**，刪除其以上所有內容。檔案首行直接從 `## 提示詞重點` 開始，**不寫 YAML frontmatter、不寫 Role、不寫絕對規則**。

| 章節 | 完整版（3500~3600 字） | 簡化版（1700~1800 字） |
|---|---|---|
| A. YAML Frontmatter | ✅ | ❌ **整段刪除** |
| B. `## Role` | ✅ 身份＋任務描述 | ❌ **整段刪除** |
| C. `## 絕對規則` | ✅ 5–6 條 | ❌ **整段刪除** |
| D. `## 提示詞重點` 10 項 | ✅ 每項 8–15 個名詞 | ✅ **保留**，每項只列 8–12 個代表性名詞 |
| E. `## 風格參考` | ✅ 3–5 位 + 說明 | ✅ **保留**，僅列名字不附說明 |
| F. `## 輸出格式` | ✅ | ✅ **保留**，一句話帶結構公式 |
| **G. `## 完整範例`** | ✅ **必備** | ❌ **整段刪除** |
| H. `## 錯誤示範` | ✅ 4–6 條 | ✅ **保留**，僅 3–5 條最關鍵的 ❌ |

## 內容深度要求

1. **名詞庫要厚且跨文化** — 完整版每章 8–15 個具體名詞 / 物件 / 形容詞，盡量涵蓋多元文化選項（亞、歐、美、非、中東、拉美），不要全部偏向單一文化。
2. **文化考據要準** — 漢服形制、和服紋樣、印度 Sari 摺法、摩洛哥 kaftan、墨西哥 huipil、奈及利亞 ankara 染布、伊斯蘭幾何花紋、原住民符號等須查證，避免張冠李戴。
3. **光影色彩可量化** — 給 K 值、底片名稱（Portra 400 / Tri-X 400 / Cinestill 800T / Velvia 50 / Ektar 100）、時段（黃金時刻 / 藍調時刻 / 魔幻時段 / 正午硬光）。
4. **鏡頭 / 媒材要對位**：
   - 攝影：焦距 / 光圈 / 機型 / 前後景模糊
   - 西方繪畫：油畫厚塗刮刀、水彩濕中濕、版畫 etching、壁畫 fresco、粉彩 pastel、丙烯
   - 東方繪畫：國畫白描渲染水墨、浮世繪 ukiyo-e、唐卡、波斯細密畫
   - 數位媒材：3D PBR 材質、Octane / Redshift 算圖、像素藝術、向量插畫
5. **錨點人物 / 作品必須真實存在** — 不確定請上網查，不可虛構。風格錨點清單盡量涵蓋多國攝影師（例：Helmut Newton 德、Mario Testino 祕、Paolo Roversi 義、Bettina Rheims 法、Annie Leibovitz 美、篠山紀信 日…）。
6. **擴展提示優於窮舉** — 與其列 30 個選項，不如列 8–12 個代表性選項＋一句「**若使用者種子提到其他類別，請自行創作合理對應**」。下游 AI 善於擴展，**不善於從窮舉清單裡隨機抓**。

## 產出前自檢清單

- [ ] **兩份檔案皆已產出**：完整版（3500~3600 字）+ 簡化版（1700~1800 字），實際字數已逐字驗算。若簡化版超字，先刪減 D 的名詞舉例、E 的說明、H 的 ❌ 條目，再驗算一次，禁止超出。
- [ ] 簡化版**首行直接從 `## 提示詞重點` 開始**，已刪除 YAML frontmatter、`## Role`、`## 絕對規則` 三段，也已刪除「完整範例」章節。
- [ ] 簡化版僅含五段：`## 提示詞重點`、`## 風格參考`、`## 輸出格式`、`## 錯誤示範`（如有其他章節須移除）。
- [ ] 簡化版 D「提示詞重點」**每項 8–12 個名詞**，未出現長串舉例。
- [ ] 完整版 `description:` 為第三人稱觸發語句、含 3–5 個關鍵字、含反例，**未與 Role 重複**。
- [ ] `Role` 為第二人稱身份定義，未混入觸發語。
- [ ] 絕對規則含該風格的**剛性禁區**。
- [ ] 提示詞重點 10 項齊全；完整版每項 8–15 個名詞。
- [ ] **#2–#8 已用 `- ` 條列格式撰寫**，每元素獨立一行；**未使用頓號（、）長串列舉**。
- [ ] **#3–#8 標題已加 `(請隨機選擇一個或自行創作)`**，每章節列表末端附「若使用者種子提到其他類別，請自行創作合理對應」的擴展提示。
- [ ] **跨文化開放**：除非主題本身就是文化限定，否則絕對規則含「種族／國籍／文化由使用者輸入推導」一條；風格錨點、服飾、場景、飲品涵蓋多元文化，未偏向單一文化。
- [ ] **絕對規則含種子擴展原則**，明示下游 AI 可自行從種子發想，不必拘泥列表。
- [ ] 風格錨點 3–5 位**真實人物 / 作品**，已查證。
- [ ] 完整範例真的示範了所有規則。
- [ ] 錯誤示範 4–6 條 ❌。
- [ ] 全文繁體中文，無簡體混入。
