# prompt-style-skill-MD

**AI 產圖提示詞「風格說明書」集合**——一套提供給「下游 AI 提示詞模型」的繁體中文風格規範檔案，由它再依使用者額外輸入生成餵給 Midjourney / Stable Diffusion / Flux / Imagen 等產圖模型的咒語。

---

## 一、專案定位

本專案**不是**直接寫給產圖模型的提示詞；本專案是**寫給「中介 AI」**的風格規範。流程如下：

```
使用者輸入「巴西嘉年華」
         ↓
[路由 AI] 依 description 載入對應 .md（system prompt）
         ↓
使用者輸入「一位 25 歲女子在桑巴大道上騰空旋轉」（額外提示詞）
         ↓
[中介 AI] 結合 .md 規範 + 額外提示詞，生成完整繁中提示詞段落
         ↓
[產圖 AI]（Midjourney / SD / Flux）依該段落產生圖像
```

每份 `<主題>_中文.md` 都是該風格的完整系統提示詞，規範了：
- 主體、服裝、動作、場景、光影、相機、構圖等十大重點
- 剛性禁區（什麼絕對不能做）
- 完整輸入→輸出範例
- 真實藝術家／攝影師／雜誌等風格錨點

## 四、使用方式

### A. 對中介 AI（GPT / Claude / 其他 LLM）
1. 將 `<主題>_中文.md` 內容貼入 **system prompt**
2. 將使用者額外提示詞貼入 **user prompt**（例：「一位 28 歲北歐混血女子，騎著 80 年代青色 Vespa」）
3. 中介 AI 會輸出符合該風格規範的完整繁中提示詞段落

### B. 對字數受限的中介 AI
改用 `<主題>_中文_簡化版.md`（≤1800 字，無 frontmatter / Role / 規則 / 範例，直接從 `## 提示詞重點` 開始）

## 五、核心設計理念

撰寫與測試過程中累積的幾個關鍵發現：

### 1. 種子 + 規則 > 列舉菜單
中介 AI **善於從種子擴展細節**（使用者寫「Vespa」就能延伸出 80 年代街景、機車排氣管聲、義大利午後光線），但**不善於從列表隨機抽選**（給它 14 個圍裙款式它永遠選第一個）。
- ✅ 系統提示應提供「規則」與「推導原則」
- ❌ 不要依賴系統提示提供「菜單供 AI 隨機選」

### 2. 首位偏見（Primacy Bias）
頓號（、）長串列舉如「A、B、C、D…」會讓下游 AI 固定抓第一個。**`PROMPT_STYLE_SKILL.md` 已強制 #2–#8 改用 `- ` 條列格式**，每元素獨立一行。

### 3. description ≠ Role（Anthropic Skill 標準）
- `description:`：第三人稱觸發語句，給**路由模型**讀，決定「何時叫我」
- `## Role`：第二人稱身份定義，給**執行模型**讀，決定「叫我之後怎麼做」
- 兩者**禁止內容重複**

### 4. 完整範例的反向錨定
中介 AI **模仿範例的傾向 >> 模仿規則**。所以完整範例不要寫「最常見」的組合，要寫**最希望被生成的代表性組合**——範例就是潛在預設。

### 5. 「指定物即唯一」剛性禁區
對「只穿一件」這類主題，明確寫死「**禁止添加使用者未指定的衣物**」並列入錯誤示範首條，可大幅降低中介 AI 自由發揮加東加西的傾向。

## 六、撰寫新風格說明書

收到主題詞（如「江南水鄉漢服少女」「賽博龐克香港」）時：

1. 讀 [PROMPT_STYLE_SKILL.md](PROMPT_STYLE_SKILL.md) 規範
2. 必要時上網查證真實攝影師 / 藝術家 / 雜誌作為風格錨點（**禁止虛構人名**）
3. 挑一份既有範本對齊章節順序與用詞節奏
4. **同時產出兩份檔案**：
   - 完整版 `<主題>_中文.md`（3500–3600 字）
   - 簡化版 `<主題>_中文_簡化版.md`（1700–1800 字，無 frontmatter/Role/規則/範例）
5. 通過產出前自檢清單
