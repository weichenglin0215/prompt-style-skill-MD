# prompt-style-skill-MD

**AI 產圖提示詞「風格說明書」集合**——一套寫給「中介 AI」的繁體中文風格規範檔案，由它再依使用者額外輸入生成餵給 Midjourney / Stable Diffusion / Flux / Imagen 等產圖模型的咒語。

> 📌 本專案**不直接寫產圖咒語**，而是製作可重複使用、跨多種文化／場景的**風格規範範本**。

---

## 一、運作流程

```
使用者輸入「巴西嘉年華」
         ↓
[路由 AI] 依 description 載入對應 .md（system prompt）
         ↓
使用者輸入「一位 25 歲女子在桑巴大道上騰空旋轉」（額外提示詞）
         ↓
[中介 AI] 結合 .md 規範 + 額外提示詞，生成完整繁中提示詞段落
         ↓
[產圖 AI]（Midjourney / SD / Flux）依該段落產生圖像
```

每份 `<主題>_中文.md` 都是該風格的完整系統提示詞，規範了：
- 主體、服裝、動作、場景、光影、相機、構圖等十大重點
- 剛性禁區（什麼絕對不能做）
- 完整輸入→輸出範例
- 真實藝術家／攝影師／雜誌等風格錨點

---

## 二、Quick Start（30 秒上手）

1. 從清單挑一份 `<主題>_中文.md`（見 §四）
2. 把該檔內容**整段貼入中介 AI 的 system prompt**
3. 在 user prompt 寫一句**簡短種子**：
   - 「**一位義大利女子在地中海別墅彎腰開烤箱**」
   - 「**北歐混血兒拳擊手在擂台上出拳的瞬間**」
   - 「**Vespa 機車上的旅人**」
4. 中介 AI 會自動推導完整場景、服飾、光影、鏡頭、構圖，輸出 500–800 字繁中提示詞段落
5. 把輸出貼進 Midjourney / SD / Flux 產圖

**字數受限的中介 AI** 改用 `<主題>_中文_簡化版.md`（≤1800 字）。

## 五、文化開放型 vs 文化限定型——怎麼挑？

| 情境 | 建議 |
|---|---|
| 使用者可能想指定不同國籍 | 🌐 文化開放型 |
| 使用者明確要日式氛圍（和服、榻榻米、清酒、京都） | 🇯🇵 文化限定型 |
| 不確定 | 🌐 文化開放型，使用者後續再用「一位日本女子，在京都町家」等關鍵字補上 |

## 六、核心設計理念

撰寫與測試過程中累積的關鍵發現：

### 1. 種子 + 規則 > 列舉菜單
中介 AI **善於從種子擴展細節**（使用者寫「Vespa」就能延伸出 80 年代街景、機車排氣管、義大利午後光線），但**不善於從列表隨機抽選**（給它 14 個圍裙款式它永遠選第一個）。
- ✅ 系統提示應提供「規則」與「推導原則」
- ✅ 列表末尾加「**若使用者種子提到其他類別，請自行創作合理對應**」
- ❌ 不要依賴系統提示提供「菜單供 AI 隨機選」

### 2. 首位偏見（Primacy Bias）
頓號（、）長串列舉「A、B、C、D…」會讓下游 AI 固定抓第一個。
- **PROMPT_STYLE_SKILL.md 已強制 #2–#8 改用 `- ` 條列格式**，每元素獨立一行
- 每個 bullet 應寫成「自完備片語」，避免依賴上下文

### 3. 三類「列表標籤」明示用途（緩解首位偏見＋鼓勵發想）
依下游 AI 對該列表的處理方式，標題後加上對應標籤：

| 標籤 | 意義 | 適用章節 |
|---|---|---|
| `(請隨機選擇一個或自行創作)` | 互斥選項，挑 1 個或自創新項 | 服裝、動作、場景、相機 |
| `(僅供參考，請自行發想)` | 例舉啟發，AI 應自由創作 | 主體、比例對比、職業範例 |
| `(請混合使用)` | 可同時組合多項 | 光影色調、材質舊化、構圖技法 |

### 4. 跨文化開放原則
- 預設**禁止套用任何單一文化**（不可永遠預設日式、歐美、東方任一）
- 種族、膚色、服飾、場景、建築、植被、氣候、光質**全部依使用者額外提示詞推導**
- 若使用者寫「義大利女人」就配地中海光；寫「俄羅斯模特」就配北方冷光；寫「奈及利亞女子」就配西非陽光與蠟染布
- 文化限定型範本（如「日本清新風格女高中」）保留作為**主題本身就限定該文化**的場合使用

### 5. description ≠ Role（Anthropic Skill 標準）
- `description:`：第三人稱觸發語句，給**路由模型**讀，決定「何時叫我」
- `## Role`：第二人稱身份定義，給**執行模型**讀，決定「叫我之後怎麼做」
- 兩者**禁止內容重複**

### 6. 完整範例的反向錨定
中介 AI **模仿範例的傾向 >> 模仿規則**。
- 完整範例不要寫「最常見」的組合，要寫**最希望被生成的代表性組合**——範例就是潛在預設
- 例如「性感裸露」的範例選**義大利托斯卡尼別墅**而非日式臥室，避免下游 AI 自動回歸日式預設

### 7. 「指定物即唯一」剛性禁區
對「只穿一件」這類主題，明確寫死「**禁止添加使用者未指定的衣物**」並列入錯誤示範首條，可大幅降低中介 AI 自由發揮加東加西的傾向。

## 七、撰寫新風格說明書

收到主題詞（如「江南水鄉漢服少女」「賽博龐克香港」）時：

1. 讀 [PROMPT_STYLE_SKILL.md](PROMPT_STYLE_SKILL.md) 規範
2. 必要時上網查證真實攝影師／藝術家／雜誌作為風格錨點（**禁止虛構人名**）
3. **預設挑跨文化開放型範本對齊**（性感裸露、只穿一件、布列松、印象派）——只有當主題本身就限定文化（含「日本」「韓國」「漢服」「Sari」等）才用文化特定範本
4. **同時產出兩份檔案**：
   - 完整版 `<主題>_中文.md`（3500–3600 字）
   - 簡化版 `<主題>_中文_簡化版.md`（1700–1800 字，無 frontmatter／Role／規則／範例）
5. 通過產出前自檢清單（含「跨文化開放」與「種子擴展」兩項驗證）
